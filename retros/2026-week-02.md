## 回顾：开始 / 停止 / 继续

### 开始做（START）
- **每次 prompt 之前先写验收标准** — 这防止了范围蔓延，也让验证变成了客观的而不是主观的。
- **用起飞前检查清单**（角色 + 输入 + 输出 + 验收） —  consistently 比随便写个 prompt 效果好。
- **把 SKILL.md 从头到尾读完** — 理解 Skill 的结构比单纯调用它学到的多得多。

### 停止做（STOP）
- **以为 agent 一次就能做对** — 我学会了把它当成初级同事：给清楚指令，检查工作，给反馈，再迭代。
- **跳过 K-S-T-A-R 里的"S"** — 一开始我直接从 K 跳到 T，没描述情境，导致任务模糊、输出不清晰。
- **过度 prompt** — 我发现自己有时候重新 prompt，但其实小的调整就够了（还省 token）。

### 继续做（CONTINUE）
- **K-S-T-A-R 循环** — 每次 R 变成新的 K，让下一轮更快。这是核心的学习机制。
- **什么都 commit** — git 历史变成了一本学习日志，回头翻很有用。
- **跟同学交换看** — Day 8 看了别人的坐标卡，给了我一些自己想不到的点子。
