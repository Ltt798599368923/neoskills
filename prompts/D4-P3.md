# D4-P3：探索文件结构

## Prompt
```
角色：你是一个代码库导航助手。
受众：一个刚接触这个 Python 项目的开发者。

输入：帮我了解一个叫 neoskills 的 Python 项目的文件结构：
- src/neoskills/（主代码）
  - adapters/（claude, openclaw, opencode 适配器）
  - cli/（命令行界面模块）
  - core/（核心功能）
  - ontology/（本体相关模块）
  - plugin/（插件系统）
  - runtime/（运行时集成）
- tests/（单元和集成测试）
- skills/（技能定义）
- agents/（agent 配置）

输出格式：
- 项目概述
- 关键目录及其作用
- 入口点在哪里

验收标准：新人看完应该知道从哪里开始读代码。
```

## Response
### neoskills 项目结构概览

**项目简介：** neoskills 是一个 Python CLI 工具，用于在多个平台（Claude、OpenClaw、OpenCode）上管理 AI agent 技能。

**关键目录：**

| 目录 | 作用 |
|------|------|
| `src/neoskills/cli/` | 命令行界面，入口是 `main.py` |
| `src/neoskills/core/` | 核心功能：配置、认证、工作区管理 |
| `src/neoskills/adapters/` | 不同 AI 平台的适配器 |
| `src/neoskills/ontology/` | 本体/技能元数据管理 |
| `src/neoskills/plugin/` | 插件系统，支持扩展 |
| `tests/` | 单元测试和集成测试 |
| `skills/` | 技能定义（SKILL.md 文件） |

**入口点：** `src/neoskills/cli/main.py` — CLI 从这里启动，想了解程序流程的话从这里开始读。
