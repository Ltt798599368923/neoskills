# D4-P3: Explore a File Structure

## Prompt
```
Role: You are a codebase navigation assistant.
Audience: A developer exploring a new Python project.

Input: Explore and explain the file structure of a Python project called "neoskills" with the following layout:
- src/neoskills/ (main source code)
  - adapters/ (claude, openclaw, opencode)
  - cli/ (command-line interface modules)
  - core/ (core functionality)
  - ontology/ (ontology-related modules)
  - plugin/ (plugin system)
  - runtime/ (runtime integrations)
- tests/ (unit and integration tests)
- skills/ (skill definitions)
- agents/ (agent configurations)

Output Shape:
- High-level project description
- Key directories and their purposes
- Entry point identification

Acceptance: A newcomer should understand where to start reading the codebase.
```

## Response
### neoskills Project Structure Overview

**Project Description:** neoskills is a Python CLI tool for managing AI agent skills across multiple platforms (Claude, OpenClaw, OpenCode).

**Key Directories:**

| Directory | Purpose |
|-----------|---------|
| `src/neoskills/cli/` | Command-line interface - entry point is `main.py` |
| `src/neoskills/core/` | Core functionality: config, auth, workspace management |
| `src/neoskills/adapters/` | Platform adapters for different AI agents |
| `src/neoskills/ontology/` | Ontology/skill metadata management |
| `src/neoskills/plugin/` | Plugin system for extensibility |
| `tests/` | Unit and integration tests |
| `skills/` | Skill definitions (SKILL.md files) |

**Entry Point:** `src/neoskills/cli/main.py` - This is where the CLI starts. Start reading here to understand the application flow.
